# Lessons Learned

## Phase 間の jj 作業環境切り替え

**パターン**: Phase 完了時は、次の Phase に進む前に必ず jj の作業環境を整理する。

**手順**:
1. 現在の作業コピーの変更を `jj describe` でコミットメッセージを付ける
2. `tasks/todo.md` の更新が含まれていることを確認する
3. 必要に応じて `jj git push` でリモートに反映する
4. `jj new develop` で develop の上に新しい空の変更を作成する（次の Phase の作業用）

**なぜ重要か**:
- Phase ごとの変更が明確に分離される
- develop ブックマーク上で直接作業するとブックマーク競合が発生する
- 「指示されるまで jj new しない」のではなく、Phase 完了 = 自動的に環境切り替えを行う

**トリガー**: Phase の全タスクが完了し、`tasks/todo.md` を更新した直後。ユーザーの指示を待たずに実行する。
